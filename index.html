API Methods:

GET Methods

Emissions Data URL: https://sustainable-transportation-tracker.onrender.com/api/transportation/emissions-data

Loads the emissions data that certain types of vehicles emit on a daily basis based on their fuel type

Parameter Description DataType
vehicleType The type of vehicle that a user can choose from String
fueltype The type of fuel that a vehicle uses between diesel or petrol Integer

Emission Reduction Tips URL: https://sustainable-transportation-tracker.onrender.com/api/transportation/emission-reduction-tips

Fetches a random tip from the database to be displayed to the user

Parameter Description DataType
tip A random tip for vehicle emission reduction will be given String

Restricted GET Methods

Weekly Vehicle Emissions URL: https://sustainable-transportation-tracker.onrender.com/api/transportation/weekly-vehicle-emissions

Fetches all the users logged emission for each week and compares it to the previous week to see if they have reduced their emissions.

Parameter Description DataType
week{week-number} Will show if they increased/decreased emission compared to previous week String

User List URL: https://sustainable-transportation-tracker.onrender.com/api/transportation/get-user-list

This would be used in conjunction with the Compare user emission, this GET will fetch all users registered so you can easily get their username to use it to compare later.

Parameter Description DataType
users A list of all the registered users. String

POST Methods

Emission Calculation URL: https://sustainable-transportation-tracker.onrender.com/api/transportation/emission-calculate

Calculates the total Emissions of a single day

Parameter Description DataType
transportationType The chosen transportation vehicle used String
distance The total distance that the user has traveled in kilometers Integer
fuelType The type of fuel used within the vehicle String

Restricted POST Methods

Register User URL: https://sustainable-transportation-tracker.onrender.com/api/transportation/register

Creates a new user account for logging in and using the Restricted API applications

Parameter Description DataType
email The user's email they want to associate their account with String
password The user's password to associate their account with String

Login User URL: https://sustainable-transportation-tracker.onrender.com/api/transportation/login

This can be used to create a JWT upon Successful login to a registered account and with the JWT you can have access to the Restricted API's

Parameter Description DataType
email The user's email associated with their account String
password The user's current password for their account String

Log Transportation Data URL: https://sustainable-transportation-tracker.onrender.com/api/transportation/log-transportation

Write the transportation data of the current week into the API database

Parameter Description DataType
transportationType The chosen transportation vehicle used String
distance The total distance that the user has traveled in kilometers Integer
fuelType The type of fuel used within the vehicle String

Compare Emission Rates URL: https://sustainable-transportation-tracker.onrender.com/api/transportation/compare-emissions

Compares a chosen weeks emission to another users for that same weeks emission and see if you had more or less than them.

Parameter Description DataType
user This will be the username of the person you want to compare to which you can get from the user list API. Integer
week The week you want to compare to String

Transportation Swapping URL: https://sustainable-transportation-tracker.onrender.com/api/transportation/transportation-swapping

Here you can select a week you have logged and change the car and fueltype to see if you would have had more or less emission, so you can use it next time to reduce emissions.

Parameter Description DataType
week The weeks emission you want to check and compare to Integer
transportationType The vehicle you want to swap to. String
fuelType The fuelType you want to swap to. String

Responses To Input:

Successful Request: Returns a 200 response code and calculates results for each function and displays it to the user.

Unsuccessful Request: Returns a 400 response code and displays to the user what the error is and/or where the error was found.

The JSON Description of Responses:

Successful Response:

Parameter Result of Method Used
error false
result An appropriate response will be generated based on the method used and displayed to the user.

Unsuccessful Response:

Parameter Result of Method Used
error true
result An appropriate error response will be displayed to the user and will alert where the error was found.
